## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

현재 바이러스 메타게노믹스 분석은 마치 **반제품을 받아서 수작업으로 완성품을 만드는 공장**과 같다.

- Hecatomb 같은 기존 도구는 "rawdata 생성기"에 불과하다
- bigtable.tsv를 받은 뒤 R/Python으로 수동 분석하고, Word로 보고서를 직접 작성한다
- 파이프라인 → 수동 분석 → 수동 보고서로 이어지는 끊긴 워크플로우
- 기존 도구들은 2020~2021년 수준의 알고리즘을 사용하며, ML 기반 바이러스 탐지, 최신 ICTV 분류체계가 반영되지 않았다
- 샘플당 100~200만원을 받는 수탁 서비스에서 이 비효율은 직접적인 비용 손실

### 1.2 목표 상황 (To-Be)

**"Raw FASTQ를 넣으면 고객에게 바로 전달할 수 있는 보고서가 나오는"** end-to-end 파이프라인.

- 최신 알고리즘(geNomad, Diamond, metaSPAdes 등)을 통합한 정확한 분석
- Pavian 수준의 동적 HTML 대시보드로 인터랙티브한 결과 탐색
- 텍스로노믹 히트맵, 다양성 분석, 통계가 포함된 Word 보고서 자동 생성
- 모듈식 설계로 도구 교체가 용이하여 시대에 뒤쳐지지 않음

### 1.3 왜 지금인가? (Why Now)

- geNomad, VirSorter2 등 ML 기반 바이러스 탐지 도구가 성숙 단계에 도달
- ICTV 2024 분류체계가 대폭 개편됨
- Nextflow/nf-core 생태계가 충분히 안정화되어 모듈식 파이프라인 구축이 용이
- 기존 도구(Hecatomb)의 Snakemake 버전 호환성 문제를 직접 경험

## 2. 사용자 페르소나

### 2.1 주요 페르소나

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "분석 전문가 J" |
| 나이/직업 | 바이오인포매틱스 분석 수탁 서비스 운영자 |
| 핵심 니즈 | 고객 샘플을 받아 최소 수작업으로 고품질 보고서 납품 |
| 불편 포인트 | 파이프라인 결과를 받은 뒤 수동으로 시각화/통계/보고서 작성에 많은 시간 소요 |
| 사용 상황 | 리눅스 서버(Threadripper PRO, 503GB RAM)에서 CLI로 분석 실행 |

### 2.2 페르소나의 하루 (시나리오)

1. 고객으로부터 곤충 바이러스 RNA-seq 샘플 2~10개 수령
2. `deepinvirus run --reads ./raw_data --output ./results --host insect` 실행
3. 파이프라인이 QC → 어셈블리 → 바이러스 탐지 → 분류 → 다양성 분석 → 시각화 → 보고서까지 자동 수행
4. `results/` 폴더에서 동적 HTML 대시보드로 결과 확인 및 탐색
5. 자동 생성된 Word 보고서를 고객 포맷에 맞게 미세 조정 후 납품

## 3. 사용자 스토리

### FEAT-1: 최신 알고리즘 통합 파이프라인 (MVP)
- [ ] 분석자로서, paired-end/single-end FASTQ를 입력하면 자동으로 QC 및 host read 제거가 되어야 한다. 왜냐하면 전처리를 매번 수동으로 하면 시간이 낭비되므로.
- [ ] 분석자로서, ML 기반(geNomad) + 상동성 기반(Diamond) 바이러스 탐지가 동시에 수행되어야 한다. 왜냐하면 novel virus까지 포함한 정확한 탐지가 필요하므로.
- [ ] 분석자로서, 최신 ICTV 2024 + NCBI taxonomy 기반으로 분류되어야 한다. 왜냐하면 5년 전 분류체계로는 논문 리뷰를 통과하기 어려우므로.
- [ ] 분석자로서, 각 단계의 도구를 config에서 교체할 수 있어야 한다. 왜냐하면 더 나은 도구가 나오면 빠르게 교체할 수 있어야 하므로.

### FEAT-2: 동적 HTML 대시보드
- [ ] 분석자로서, Pavian 수준의 인터랙티브 HTML 대시보드를 받아야 한다. 왜냐하면 결과를 동적으로 필터링/탐색해야 고객 질문에 빠르게 답할 수 있으므로.
- [ ] 분석자로서, 텍소노믹 히트맵, Sankey 다이어그램, 다양성 플롯이 포함되어야 한다.

### FEAT-3: Word 보고서 자동 생성
- [ ] 분석자로서, 분석 결과가 포함된 Word 보고서가 자동으로 생성되어야 한다. 왜냐하면 수동 복사-붙이기가 전체 시간의 큰 비중을 차지하므로.
- [ ] 분석자로서, 보고서에 그림, 테이블, 해석 텍스트가 포함되어야 한다.

## 5. 범위 관리

### 5.1 MVP (반드시 포함)
- FEAT-1: QC → 어셈블리 → 바이러스 탐지(ML+상동성) → 분류(ICTV 2024) → 다양성 분석
- FEAT-2: 동적 HTML 대시보드 (히트맵, Sankey, 다양성 플롯)
- FEAT-3: Word 보고서 자동 생성

### 5.2 옵션 (시간 허용 시)
- Phage-host prediction (iPHoP)
- MAG (Metagenome-Assembled Genome) 복원
- Long-read (Nanopore/PacBio) 지원

### 5.3 Non-goals (명시적 제외)
- 웹 기반 SaaS 서비스: CLI 파이프라인에 집중
- 실시간 모니터링: Nextflow 자체 리포트로 대체
- 16S rRNA 분석: 바이러스 메타게노믹스에 한정

### 6.2 Top 5 리스크 보드

| # | 가정 | 리스크 | 완화 | 트리거 | 대응 |
|---|------|--------|------|--------|------|
